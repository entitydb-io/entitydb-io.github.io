## Remarks

The transaction builder does not attempt to auto-load entities. If an entity exists but has not been loaded into the transaction builder, the [`EntityAlreadyCreatedException`](./EntityAlreadyCreatedException.md) will not be thrown. Howevever, an [`OptimisticConcurrencyException`](./OptimisticConcurrencyException.md) will be logged if an attempt is made to commit the transaction, and ITransaction will return false.
